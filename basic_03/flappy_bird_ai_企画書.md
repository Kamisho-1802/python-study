## 1. プロジェクト概要

### 目的
Flappy Bird ゲームを Python で自作し、**NEAT アルゴリズム（進化的ニューラルネットワーク）** を使って AIに自動プレイを学習させる。世代を重ねるごとに賢くなる AI の成長をリアルタイムで観察できる、インタラクティブな機械学習体験アプリを構築する。

### コンセプト
> 「AIが失敗を繰り返しながら、自力で上手くなっていく様子を目で見る」

### NEATとは？
**NeuroEvolution of Augmenting Topologies** の略。
進化的アルゴリズムでニューラルネットワークの**構造と重みを同時に進化**させる手法。強化学習の報酬設計が不要で、「生き残ったものが次世代に遺伝子を渡す」という自然選択のモデルが直感的にわかりやすい。

## 3. ゲーム仕様

### 3-1. ゲームルール（Flappy Bird 準拠）
- 鳥は常に重力で落下する
- スペースキー（または AI の判断）で羽ばたき、一時的に上昇する
- 画面を左から右へ流れるパイプの隙間を通り抜ける
- パイプに当たるか画面外に出ると即死

### 3-2. 画面構成

```
+------------------------------------------+
|  GENERATION: 5    SCORE: 42    ALIVE: 8  |
|                                           |
|      [鳥]                                |
|              ||      ||                  |
|              ||      ||   <-- パイプ     |
|              ||      ||                  |
|                                           |
+------------------------------------------+
```

### 3-3. パラメータ設定

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| 画面サイズ | 600 × 800 px | ゲームウィンドウ |
| 重力加速度 | 0.5 px/frame | 鳥の落下速度 |
| ジャンプ力 | -10 px/frame | 羽ばたき時の上昇速度 |
| パイプ速度 | 5 px/frame | パイプの移動速度 |
| パイプ間隔 | 150 px | 上下パイプの隙間 |
| FPS | 60 | フレームレート |

## 4. AI 設計

### 4-1. ニューラルネットワークへの入力（センサー情報）

AIが判断に使う情報は以下の5つ：

```
入力ノード
├── 鳥の Y 座標（現在の高さ）
├── 鳥の 縦速度（上昇中か落下中か）
├── 次のパイプまでの 水平距離
├── 次のパイプ（上）の Y 座標
└── 次のパイプ（下）の Y 座標

          ↓ 隠れ層（NEATが自動設計）

出力ノード
└── ジャンプするか否か（0 or 1）
```

### 4-2. 適応度関数（Fitness Function）
AIの「上手さ」を数値化するルール：

```python
fitness += 0.1          # 毎フレーム生き延びるたびに加算
fitness += 5.0          # パイプを1本通過するたびに加算
fitness -= 1.0          # 死亡時にペナルティ（オプション）
```

生き延びるほど、パイプを多く通るほど高スコアになる。

### 4-3. NEATの進化プロセス

```
第1世代（ランダムな脳）
  ↓ ほぼ全員すぐ死ぬ
生き残った個体の遺伝子を選別
  ↓ 突然変異 + 交叉
第2世代（少しだけ賢い脳）
  ↓ ...繰り返す
第N世代（パイプを無限に通れる脳）
```

## 7. NEAT 設定ファイル（config-feedforward.txt）

```ini
[NEAT]
fitness_criterion     = max
fitness_threshold     = 1000
pop_size              = 50
reset_on_extinction   = False

[DefaultGenome]
# ネットワーク構造
num_inputs            = 5
num_outputs           = 1
num_hidden            = 0

# 活性化関数

[DefaultSpeciesSet]
```

## 8. 期待される学習曲線

| 世代 | 期待される挙動 |
|------|---------------|
| 第 1〜3 世代 | ほぼ全員がすぐ墜落。偶然生き残る個体が出始める |

## 10. 学習のポイント・つまずきやすいポイント

### ✅ うまくいくコツ
- `pop_size`（個体数）は最初 **50程度** が適切。多すぎると重くなる
- フィットネスの加算タイミングをフレーム単位にすると収束が速い
- ゲームの FPS を下げると（例：30）学習が速く進む

### ⚠️ よくある失敗
- **全世代が同じ行動をする** → `weight_mutate_rate` を上げる
- **一向に学習が進まない** → フィットネス関数の設計を見直す
- **pygame ウィンドウが固まる** → イベントループが正しく回っているか確認
